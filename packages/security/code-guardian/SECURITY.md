# Security Policy

## Reporting Security Vulnerabilities

**DO NOT** open public issues for security vulnerabilities.

Instead, please report them privately via:
- Email: security@example.com (replace with actual email)
- GitHub Security Advisories: https://github.com/consigcody94/code-guardian/security/advisories/new

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if available)

## Response Time

- **Critical**: 24 hours
- **High**: 48 hours  
- **Medium/Low**: 1 week

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | ✅                 |
| < 1.0   | ❌                 |

## Security Best Practices

When using Code Guardian:
1. Run in CI/CD pipelines
2. Fail builds on critical issues
3. Review findings before deployment
4. Keep Code Guardian updated
5. Use `--fail-on` flag appropriately
